# Roadmap

This roadmap tracks project milestones only.

## Documentation v1.0

Goal:

- Complete the AI development documentation set required before RC7 analysis and refactoring.

Outcome:

- Codex can read project rules, architecture, data contracts, coding standards, design standards, test standards, review standards, Git workflow, and active task instructions before touching code.

## RC7

Goal:

- Refactor the GitHub Pages frontend without changing backend or data contracts.

Focus:

- Dashboard clarity.
- Table renderer consistency.
- Modal and form safety.
- Design system cleanup.
- CSS duplication cleanup.
- Frontend module responsibility clarity.

Exit Criteria:

- Apps Script API unchanged.
- Spreadsheet schema unchanged.
- API endpoint unchanged.
- JSON response format unchanged.
- Frontend behavior validated.
- Git diff reviewed.
- No commit or push unless explicitly requested.

## RC8

Goal:

- Improve product workflows after RC7 stabilizes frontend structure.

Possible focus:

- Better review flow for opinions and decisions.
- More useful framework browsing.
- Improved roadmap visibility.

## RC9

Goal:

- Prepare v1.0 release candidate quality.

Possible focus:

- Regression hardening.
- Accessibility pass.
- Documentation alignment.
- Deployment checklist.

## v1.0

Goal:

- Release a stable Research Lab experience for team use.

Expected state:

- Clear frontend structure.
- Stable data contract.
- Reliable dashboard and table experience.
- Documented AI-assisted maintenance workflow.
